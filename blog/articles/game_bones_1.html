Game Bones: Basic Movement

I codified goal #1 of my little project as follows:

"A character that moves in response to player input. Movement should consist of several states with a speeding up phase, constant top speed, and slowing down phase (See Prince of Persia)."

I’ve successfully implemented this both in Unity and using my own C code + SDL2. As you can see, they are pretty close.

Unity:

Game Bones:

There’s a basic state machine for the player with animations tied directly to the states (idle, walk, run, slow down) and some basic input handling. Currently there’s no gravity, collision detection, etc. In Unity I’m just calling transform.Translate() and keeping track of my own velocity and acceleration variables.

This made it very easy to “port” from Unity into the C project because, of course, I’ll always have to keep track of everything myself in the C project. I expect parity between the projects to become more difficult from now on. The next goal is jumping and some collision detection with static obstacles. I plan to leverage Unity’s physics system for that, but I also don’t plan to implement anything like a full physics simulation in Game Bones. I don’t need one for this.
